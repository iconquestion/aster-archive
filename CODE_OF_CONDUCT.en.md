# Code of Conduct

## Purpose

`Aster Archive` is a personally maintained Web puzzle project, and also a public space for discussion and collaboration. Whether you are here to play, ask questions, give feedback, discuss design choices, or contribute code and documentation, the goal is to keep this space respectful, patient, and constructive.

This code of conduct is not meant to make discussion stiff or overly formal. Its purpose is simply to reduce unnecessary hostility, humiliation, spoiler-driven disruption, and unproductive conflict, so that conversations can stay focused on the project itself.

## Expected Behavior

The following kinds of participation are welcome here:

- respectful, sincere, and issue-focused discussion
- patience toward people with different levels of experience
- constructive feedback on code, documentation, design, and gameplay
- explaining criticism with context, reasons, or improvement ideas when possible
- respecting the personal voice, subjective design choices, and experimental nature of the project
- avoiding unnecessary public spoilers when discussing puzzle content
- respecting the time, energy, and boundaries of the author and other participants

## Unacceptable Behavior

The following behaviors are not acceptable in this project:

- personal attacks, insults, harassment, abuse, or deliberate provocation
- discriminatory language based on identity, background, gender, sexual orientation, region, language, religion, or experience level
- repeated intimidation, threats, stalking, or targeted hostility
- trying to embarrass others, especially newer participants or people asking questions
- publicly dumping answers, hidden paths, or core puzzle solutions in ways that clearly damage the play experience
- inflammatory off-topic arguments, spam, or malicious promotion
- publishing someone else's private information, private communication, or sensitive details without permission
- using “feedback” as a pretext for hostile venting instead of communicating about the actual issue

## Spoilers and Discussion Boundaries

This is a puzzle project, so how we discuss things matters.

Discussion about gameplay, difficulty, hint design, implementation, and level experience is welcome, but please be mindful of what you reveal in public. In particular, avoid putting full answers, next-level paths, or critical hidden clues directly into Issue titles, Discussion titles, PR titles, or other highly visible places. If a detailed solution really needs to be discussed, please make the spoiler boundary clear and keep the wording considerate.

The project welcomes discussion. It does not welcome casually ruining someone else’s first experience.

## Scope

This code of conduct applies to public interactions related to the project, including but not limited to:

- GitHub Issues
- GitHub Discussions
- Pull Requests
- commit messages and code review conversations
- project-related email communication

If behavior happens outside the repository but is clearly directed at project participants and has a meaningful impact on the project environment, it may also be considered within scope.

## Enforcement

This project is personally maintained, so interpretation and enforcement of this code of conduct are handled by the maintainer.

Depending on the situation, responses to inappropriate behavior may include:

- a reminder or warning
- a request to edit comments, titles, descriptions, or other public content
- hiding, locking, or deleting inappropriate discussion content
- closing related Issues, Discussions, or Pull Requests
- declining further interaction or contributions
- blocking participants when necessary

The priority will be maintaining a workable discussion environment, not escalating conflict.

## Reporting

If you experience or observe clearly inappropriate behavior in project-related interactions, you can contact the maintainer at:

- Email: `mojavenight@qq.com`

If the matter is sensitive, please consider contacting privately instead of escalating the conflict in public first.

## Final Note

This project welcomes people with different backgrounds, skill levels, and communication styles, as long as basic mutual respect is maintained.

You do not need to agree with every design choice, and you do not need to sound perfectly gentle at all times. But your words should help move things forward, rather than turning the conversation itself into the next problem.
