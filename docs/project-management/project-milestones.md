# 项目里程碑

本文档用于记录 `Aster Archive` 第一阶段项目的里程碑。

本文档将若干连续开发活动归并为适合项目计划使用的阶段节点。

## 里程碑口径

第一阶段范围以 [项目范围说明](./project-scope.md) 为准，即：

第 `01` 关至第 `24` 关的完整主线，作为当前主要项目范围。

第 `25` 关及之后的内容可作为后续扩展探索，但不纳入本阶段必交里程碑。

## 详细说明

### M0 项目立项与仓库初始化

时间节点：`2026-03-24`

#### 目标

- 建立项目仓库
- 完成最基本的项目说明
- 确立授权与公开方式

#### 对应完成情况

- 项目初始化完成
- 首次提交 `README` 与许可证
- 仓库基础结构建立

#### 里程碑意义

这一阶段代表项目从想法进入可管理状态，具备继续累积内容和代码的最小基础。

### M1 主线内容原型形成

时间节点：`2026-03-24` 至 `2026-04-04`

#### 目标

- 形成可游玩的 Web 解谜核心玩法
- 逐步建立“通过前后端交互寻找下一关入口”的基本节奏
- 让项目不再只是概念，而是具有连续体验的原型

#### 对应完成情况

- 首页、起始页和多关卡内容逐步落地
- 解谜方式开始覆盖页面源码、资源文件、请求响应、接口行为等不同切面
- 前后端混合玩法的主线结构成形

#### 里程碑意义

这一阶段完成后，项目已经具备“可玩”的核心价值，不再只是零散页面集合。

### M2 仓库公开与主线扩展至 24 关

时间节点：`2026-04-05` 至 `2026-04-06`

#### 目标

- 对外公开仓库
- 持续扩展关卡内容
- 将主线推进到足以作为第一阶段正式范围的体量

#### 对应完成情况

- 仓库公开发布并进入连续更新
- 第 `22` 关、第 `23` 关、第 `24` 关陆续加入
- 首页明确对外展示“已开放至第 24 关”

#### 里程碑意义

这一节点标志着第一阶段主线内容达到正式范围要求，后续计划据此以“1-24 关”为边界进行编制。

### M3 文档体系与协作规范完善

时间节点：`2026-04-06` 至 `2026-04-11`

#### 目标

- 让项目具备可维护、可协作、可说明的外围文档
- 完善玩法实现之外的工程背景说明

#### 对应完成情况

- 完善 `README.md`
- 完成 `docs/architecture/system-architecture.md` 架构说明
- 建立贡献指南、行为准则、安全策略、PR 模板等文档
- 持续维护题解、背景设定和页面设计说明

#### 里程碑意义

这一阶段完成后，项目从“能运行的个人仓库”进一步提升为“别人可以理解和参与的项目”。

### M4 工程化与测试体系升级

时间节点：`2026-04-07` 至 `2026-04-25`

#### 目标

- 提高后端与协议类关卡的可验证性
- 降低后续维护成本
- 为持续演进提供更稳定的工程底座

#### 对应完成情况

- 测试体系使用 Jest 与 Supertest
- 维护多个关键关卡测试
- 接入 GitHub Actions
- 增加日志轮转
- 引入 ESLint、Prettier 与提交前自动格式化
- 对服务入口进行拆分并补充错误处理

#### 里程碑意义

这一阶段让项目的关注点从“把内容做出来”扩展到“把内容稳定地维护下去”。

### M5 第一阶段范围确认与计划记录

时间节点：`2026-04-25`

#### 目标

- 明确第一阶段正式范围
- 记录项目范围、里程碑与项目进度
- 为后续扩展保留清晰边界

#### 对应完成情况

- 明确第一阶段以第 `01` 至第 `24` 关为主范围
- 将第 `25` 关及之后内容定位为后续扩展方向
- 形成项目范围、里程碑、项目进度三份独立文档

#### 里程碑意义

这一阶段使项目具备了完整、明确的阶段计划记录。
