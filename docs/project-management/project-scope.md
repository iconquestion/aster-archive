# 项目范围说明

本文档用于记录 `Aster Archive` 第一阶段的正式项目范围，作为需求、里程碑、进度和验收判断的边界依据。

本阶段范围不是对整个项目的永久承诺，而是对已经形成稳定主线、可对外说明、可用于计划管理与验收的部分进行收束。

## 文档目的

- 明确第一阶段项目到底要交付什么
- 将“已完成主线”和“后续扩展方向”区分开
- 为里程碑、项目进度、验收口径提供统一边界

## 第一阶段正式范围

第一阶段项目范围定义为：

`Aster Archive` 自起始页开始，到第 `26` 关为止的完整 Web 解谜主线。

这一范围包含：

- 首页、起始说明页与更新页
- 第 `01` 关至第 `26` 关的可游玩关卡体验
- 支撑主线解谜所需的题面、线索、提示、叙事文本与静态物料
- 范围内关卡所需的页面、资源、接口与必要运行支撑
- 支撑玩家理解玩法与推进主线的公开说明

## 范围内交付物

第一阶段范围内的交付物主要包括以下内容。

### 1. 玩家可游玩的主线内容

- 首页、起始说明页与更新页
- 第 `01` 关至第 `26` 关的页面、线索、提示与静态资源
- 支撑各关卡解谜过程所需的页面交互、接口和协议能力
- 维持题面叙事连续性的必要文本、命名和世界观材料

### 2. 后端与运行支撑

- 范围内关卡所需的后端接口与协议支撑
- 支撑主线稳定访问所需的基础运行能力
- 支撑玩家理解玩法和维护者复核主线的必要文档

## 范围边界

为了避免项目计划失焦，第一阶段仅以“主线已开放至第 26 关”为完成边界。

这意味着：

- 是否存在第 `27` 关及之后相关代码，不影响第一阶段范围定义
- 第 `27` 关及之后的后续章节、新玩法或大规模世界观扩展，不计入第一阶段必交内容
- 对第 `01` 至第 `26` 关之外的试验性实现，可以继续保留在仓库中，但不作为第一阶段验收前提

## 明确不纳入第一阶段验收的内容

以下内容不纳入第一阶段正式验收范围：

- 第 `27` 关及之后的关卡
- 仅处于实验、打磨或局部重构中的新章节
- 更大规模的内容运营、社区化功能或账号体系
- 将项目改造为通用平台、编辑器或关卡生产工具

## 第一阶段完成判定

若从项目计划视角判断第一阶段是否达到预期，至少应满足以下条件：

- 首页与起始页可正常引导玩家进入主线
- 第 `01` 关至第 `26` 关构成一条连续可游玩的主线链路
- 范围内需要后端参与的关键关卡具备稳定服务能力
- 范围内需要状态、会话或持久化支撑的关卡具备可复核行为
- 项目具备支撑玩家理解与维护者复核的基础说明、题解和设计资料
- 对外描述与内部计划口径一致，均以“已开放至第 26 关”为阶段边界

## 后续扩展原则

第一阶段范围冻结后，后续扩展建议遵循以下原则：

- 第 `27` 关及之后的新关卡按新阶段或新版本单独规划
- 不为了追求章节数量而频繁改写既有主线边界
- 新内容如影响架构、测试或叙事节奏，应同步更新里程碑与进度文档
