# Aster Archive 背景设定总表

本文档用于整理 Aster Archive 的背景设定框架，供后续关卡、页面文案、道具文本、房间命名、制度细节统一参考。

阅读建议：

- 先看"稳定主设定"：这是默认长期成立、可反复使用的基础框架。
- 再看"可变深层设定"：这是目前保留弹性的深层背景，不要求一次写死。
- 最后看"创作约束"：这是写页面、文书、系统、关卡说明时的统一口径。

## 一、稳定主设定

### 1. 项目核心

- 核心定位
  - Aster Archive 的舞台，是一座曾长期真实运转、制度严密、文书繁复的古老档案馆。
- 外部观感
  - 对玩家呈现为一座古典、真实、厚重，但又隐约残留现代甚至超前技术痕迹的档案馆。
- 情绪目标
  - 玩家首先应感受到未知、神秘、厚重与轻微不安，意识到自己正在进入一座有历史、有制度、也有隐秘面的建筑。
  - 这里的节奏变化，仍然属于同一条主线探索内部的起伏，而不是把"紧张主线"和"轻松支线"分开处理。
  - 所谓"紧张"也不应被简单理解成持续压迫，而应拆分为两种互相配合的驱动力：面对未知空间、陌生物件和新鲜设定时产生的好奇感与新鲜感，以及面对无法完全探明的神秘现象、缺失信息和略带危险意味的内容时产生的轻微恐惧感。
  - 这两种情绪应相辅相成。好奇推动玩家继续前进，恐惧提醒玩家这座建筑并不安全、也并非一切都能被顺利解释。
  - 因此整体体验应当是：以未知带来的探索欲为底，以轻微恐惧维持深层张力，再用古典、浪漫、音乐性与人情痕迹提供适时的缓和与呼吸感，通过张弛有度的节奏变化提升持续游玩体验。
  - 例如，玩家发现一件前所未见但设定精巧的旧物时，应首先感到好奇；而当玩家读到 [2013-12-21.txt](../../public/08-c2x8m5q9nv/stack/restricted/914/2013-12-31.txt) 这类略带恐怖意味、又无法完全解释的材料时，则应感到轻微但真实的恐惧。
- 叙事重点
  - 玩家通过档案馆留下的各种痕迹继续探索，包括空间、文书、制度、页面、注释、目录、系统残留与接口痕迹，并据此寻找下一步入口。
- 重要补充
  - Web 题目中的页面、资源、接口、协议与系统细节，是背景设定在网页谜题中的映射方式之一，而不是要求把整个世界观都写成"信息层探险"。

### 2. 时间与气质

- 创作参考年代
  - 以 18 世纪末至 19 世纪末的欧洲古典建筑风格、书面文体与生活气息为主要参考。
- 实际呈现状态
  - 不是纯古典奇幻，也不是纯现代废墟，而是"古典制度长期存活，并在局部吸纳了现代与超前技术"的状态。
- 主气质结构
  - 主体仍然是未知、厚重、神秘、安静、略带压迫感的旧档案馆探索。
  - 同时允许局部出现较温柔的古典美感，例如音乐、花体字、私人书信、演奏空间、纪念性物件与带人情味的旧痕迹。
  - 这些较轻盈的部分不是为了削弱主线神秘感，而是为了给玩家提供呼吸感、恢复感和情绪上的缓冲。
- 推荐历史分层
  - 初建期：贵族或学术机构气质明显，重收藏与分类。
  - 扩张期：形成完整馆务制度，房间、部门、报刊、登记系统成熟。
  - 技术转向期：引入电话、打字机、广播、供电设施、电子终端，以及少量无法完全以现实技术解释的馆内设备。
  - 遗留期：古老建筑、纸本文书、近现代系统与少量超前技术残迹并存，玩家接触到的是这些层次叠压后的残留运行面。

### 3. 地域与审美参考

- 文化基底
  - 以欧洲古典馆藏机构为主要参考，但不绑定现实中的具体国家、王朝或宗教体系。
- 推荐外观元素
  - 高窗
  - 长廊
  - 木制目录柜
  - 黄铜铭牌
  - 厚重窗帘
  - 石质或木质地面
  - 玻璃陈列柜
  - 少量嵌入其中但风格克制的技术装置
- 推荐书面气质
  - 正式
  - 克制
  - 讲究格式
  - 略有陈旧感
  - 礼貌但疏离
- 可复用的气质簇
  - 除了"厚重馆藏机构"这一主气质外，也允许局部出现更柔软的支线氛围。
  - 当前已较稳定成立的一组是：`古典`、`音乐`、`浪漫`、`私人文本`、`低照度`、`旧纸张`、`钢琴`、`手写补记`。
  - 这组元素适合出现在音乐厅、附属演奏空间、私人留存物、未寄出的信件、尾部补记、手写附页等场景中。
  - 使用时应保持含蓄、克制、略带迟来感，像旧时代遗留的私人情绪，而不是直白的恋爱叙事。

### 5. 科技与信息系统

#### 5.1 科技设定原则

- 总体定位
  - 科技不是世界观的表层主题，而是档案馆在部分历史阶段、部分部门和部分设施中留下的特殊能力。
- 使用原则
  - 少用、慎用，非必要不用。
  - 优先表现为"基于人类已有发明，但在少数方向上被大幅强化"的技术。
  - 科技元素应服务于谜题、空间气质或机构可信度，而不是喧宾夺主。
- 推荐风格
  - 看上去仍然像馆内设备、研究设施、安保系统、供能装置、识别装置，而不是完整赛博朋克体系。
- 推荐示例
  - 远超常规功率与稳定性的馆内发电设备。
  - 基于生物特征甚至 DNA 识别的门禁装置。
  - 能在特定条件下自动校对、标记或追踪卷宗的管理设备。
  - 稳定运行时间异常漫长的中继系统、监测终端或恢复装置。
- 不建议写法
  - 未来兵器
  - 全自动 AI 社会
  - 夸张科幻城市级设施
  - 每关都依赖超前科技成立

#### 5.2 普通信息系统设定

- 基础定位
  - 馆内确实存在过一批常规信息平台系统，它们很多只是普通的信息化管理产物。
- 推荐系统类型
  - 馆藏检索系统
  - 访客登记系统
  - 部门内部登录后台
  - 设备维护后台
  - 权限审批与恢复系统
  - 订阅与投递记录系统
- 推荐技术观感
  - 同时存在 C/S 与 B/S 的遗留。
  - 同时存在本地终端、内网页面、简单 API、静态帮助文档、配置页与恢复页。
  - 登录、密码、权限分级、账户锁定、日志、会话等概念都是正常存在的。
- 使用原则
  - 这些系统首先应当让人感觉"像人类机构真的会部署的普通系统"。
  - 少数关键系统可以再与超前科技或深层设定发生连接。
  - 不要把每台机器都写成不可理解的特异产物。

### 6. 制度

- 基础制度
  - 借阅登记制度
  - 房间访问权限制度
  - 特藏隔离制度
  - 馆内报刊订阅与人工投递制度
  - 机密材料离线流转制度
  - 馆内信息平台登录与权限制度
  - 文书编号与归档制度
  - 封存与停用公告制度
  - 值班与轮班制度
  - 设备授权与门禁识别制度
- 默认细节
  - 不同身份可访问不同区域。
  - 总目录不一定等于实际投递记录。
  - 旧记录允许保留边注与历年补记。
  - 人工抄录版本与正式登记版本可能并存。
  - 重要材料可能存在"公开目录项"与"限制阅览项"两套描述。
  - 一部分材料因保密、稳定性或访问习惯等原因，不直接接入馆内系统，而继续保留纸本、人工投递或封签流转。
  - 某些部门即使已经具备技术终端，也仍偏好保存纸质副本、手写附注与签收记录。
  - 馆内的信息平台并不神秘，很多只是常规的检索、登记、登录、调度、恢复与权限管理系统。

#### 7.3 房间名称与作用

- 主览大厅
  - 访客最先接触的公共空间，展示馆的历史、制度与总导览。
- 公共档案区
  - 开放检索与基础借阅，玩家最容易接触目录、索引、分类信息。
- 展示长廊
  - 陈列复制品、摘要、纪念性材料，适合做"表面叙述与真实记录不一致"。
- 管理办公室
  - 馆务规划、展出安排、内部协调，适合藏路线索和次级文书。
- 阅览室
  - 正式查阅区域，适合借阅单、座位编号、时间限制、抄写规则。
- 编目室
  - 整理新到馆材料、修订目录、编号变更，适合做版本差异和索引谜题。
- 抄录房
  - 誊写摘要、制作副本、整理来函，适合制造"原件与抄本差异"。
- 特藏柜 / 特藏室
  - 保存高价值或限制访问材料，适合低频异常设定。
- 地下书库
  - 长期保管旧卷宗、地方志、散页手稿，适合阴暗但不一定超自然。
- 修复室
  - 纸张修补、装订、污损处理，适合物理材料类线索。
- 报刊订阅台
  - 发布内部新闻、订阅名册、人工投递，适合做信息密集的文书谜题，也适合暗示"不是所有材料都适合接入系统"。
- 来函登记处
  - 记录外来信件、捐赠、通知、公文往来，适合做身份与时间线线索。
- 安保值班室
  - 保留巡查、门禁、设备记录，适合做系统残留与监控类关卡。
- 运维恢复终端室 / 中继控制点
  - 偏后期技术层，解释现代接口、恢复流程、配置残留。
- 系统维护室
  - 处理馆内终端、索引设备、网络节点与权限配置，适合承接 Web 题目中的接口、配置、注释与残留文件。
- 馆内信息平台主管室
  - 维护普通检索平台、后台登录、部门账号、权限分组与前后台页面，适合为常规 Web 关卡提供自然语境。
- 识别与授权室
  - 负责门禁、生物识别、凭证签发与权限绑定，适合放置少量超前技术设定。
- 供能室
  - 维持馆内某些异常稳定或功率过高的设备运行，适合暗示"超常但不外放说明"的科技背景。
- 观测记录室
  - 保存设备状态、运行日志、环境读数与异常报告，适合承接技术层与文书层之间的过渡。
- 馆内医务室
  - 平时用于馆员身体状况处理，少数异常事件时可作为警告性地点。
- 国际宾客厅
  - 外宾接待、多语言说明、礼仪性空间，适合拓展"国外机构感"。
- 音乐厅 / 小演奏厅
  - 用于馆内小型演奏、朗读、纪念活动或内部聚会，适合承接钢琴、乐谱、私人纸面材料、演出流程记录与尾部补记。
- 封存库房
  - 停用设备、撤下展品、旧批次材料集中保管。
- 独立库房
  - 极少开放、单独编号、强限制材料所在处。

## 二、可变深层设定

### 9. 异常设定

- 基础定位
  - 这是副设定，不是表层常驻主题。
- 使用目的
  - 强化危机感。
  - 制造刺激感。
  - 给关键关卡升压。
- 推荐呈现方式
  - 异常批注
  - 借阅记录不一致
  - 姓名、编号、日期被轻微改写
  - 材料对阅读者产生不适或被动记录
- 使用边界
  - 不要频繁跳脸。
  - 不要让每个谜题都归因于异常。
  - 不要压过"真实旧档案馆制度感"。

### 10. 隐形深层背景

- 当前推荐方向
  - 整个档案馆可能长期受某种外部因素影响。
  - 这种影响一开始并不表现为显眼灾难，而是缓慢、局部、难以定性的异化。
  - 馆内人员、文书、制度、设备和空间，都会在长期作用下逐渐出现偏移、失真、异常依赖或认知层面的变化。
- 高层处理方式
  - 高层并未公开承认这一问题，而是试图通过封存、裁撤、停用、权限收紧、部门合并、限制接触和逐步闭馆来掩盖影响范围。
  - 因此玩家后来看到的大量"停用""封存""暂停更新""不再开放"，不一定只是普通衰败，也可能是被动收缩后的结果。
- 与现有内容的兼容方式
  - 这层背景可以把"超前科技残留""异常档案""部门停摆""闭馆痕迹"自然串联起来。

## 三、创作约束

### 11. 文风分层

- 馆内正式文书层
  - 应保持正式、克制、安静、略旧。
  - 以制度、流程、记述、说明为主。
  - 通过细节显露时代感。
  - 即使涉及音乐、私人情感或纪念性事件，也优先通过流程单、补记、附页、夹带纸条等间接方式呈现。
- 面向玩家的页面表层
  - 玩家默认是现代中文互联网语境下的年轻人，具备天然好奇心与探索欲，这一点可以直接作为推动事件发展的默认前提。
  - 因此关卡介绍、提示、按钮文案、轻量吐槽中，可以适度使用现代中文互联网表达、轻微玩梗和流行语，以吸引新玩家并降低进入门槛。
  - 这一层口吻尤其适用于靠后的关卡、转场、提示、引导和故意做出反差感的页面。
  - 这一层口吻不等于馆内正式文书口吻，二者应明确分层。
- 深层失衡层
  - 只在少数关卡轻微失衡。
  - 从措辞、注释、删改痕迹里露出危险感。
- 私人抒情层
  - 仅在少数关卡作为馆内正式材料之外的补充层出现。
  - 关键词可参考：`浪漫`、`古典`、`音乐`、`私密`、`迟到的告白`、`未寄出的信`、`旧纸与手写感`。
  - 这一层应服务于"人曾在这里生活与停留"的真实感，而不是把整体叙事改写成爱情故事。

### 12. 节奏设计原则

- 张弛关系
  - 整体流程不宜持续维持单一的压抑、惊悚或高警觉状态。
  - 在受限区、封存区、异常设施、权限系统带来的紧张感之间，应穿插少量更舒展、更可亲近的空间与文本。
- 轻松段落的作用
  - 这类段落可以通过古典审美、音乐意象、私人情书、纪念性物件、旧日生活痕迹等方式出现。
  - 它们的作用不是"搞笑"或"脱戏"，而是让玩家感到这座建筑并非只有危险与冷硬制度，也曾经承载过审美、情感与日常生活。
- 玩家体验目标
  - 玩家应在"继续深入的紧张感"和"短暂停留的轻松愉悦感"之间反复切换。
  - 这种切换能帮助调节节奏、降低审美疲劳，并放大后续神秘与异常内容的冲击力。

### 13. 总体禁忌

- 不要强行写成中式单位口吻。
- 不要把馆内正式文书频繁写成玩梗口吻。
- 不要把所有层级都写成现代口语。
- 不要一上来就高强度超自然。
- 不要把所有房间都写成 SCP 收容室。
- 不要让科技设定压过古典馆藏机构本身。
- 不要让轻松、浪漫元素泛滥到削弱整体探索感与未知感。
