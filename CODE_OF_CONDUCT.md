# 行为准则

## 目的

`Aster Archive` 是一个个人维护的 Web 闯关游戏项目，也是一处公开交流与协作的空间。无论你是来游玩、提问、反馈、讨论设计，还是提交代码和文档贡献，都希望这里能保持基本的尊重、耐心与善意。

这份行为准则的目标不是把交流变得生硬，而是尽量减少不必要的攻击、羞辱、剧透式破坏和消耗，让项目讨论能围绕问题本身展开。

## 我们希望看到的行为

在这个项目中，欢迎以下类型的交流与参与：

- 友善、真诚、就事论事的讨论
- 对不同经验水平的参与者保持耐心
- 针对代码、文档、设计和玩法提出具体且有建设性的反馈
- 在指出问题时尽量附带背景、理由或改进建议
- 尊重项目的个人表达、主观设计和实验性质
- 在讨论关卡时尽量避免无必要地公开剧透
- 理解并尊重作者与其他参与者的时间和精力边界

## 不可接受的行为

以下行为不适合出现在这个项目中：

- 人身攻击、侮辱、嘲讽、辱骂或恶意挑衅
- 基于身份、背景、性别、性取向、地域、语言、宗教、经验水平等的歧视性言论
- 持续性的骚扰、纠缠、威胁或恐吓
- 故意让他人难堪，尤其是针对新手或提问者的羞辱式表达
- 明知会破坏游玩体验，仍在公开区域大规模泄露答案、隐藏路径或关键谜底
- 发布与项目无关的引战内容、垃圾信息或恶意广告
- 未经允许公开他人的隐私、私人通信内容或敏感信息
- 以“反馈”为名进行情绪化发泄，而非围绕问题本身沟通

## 关于剧透与讨论边界

这是一个解谜项目，因此“怎么讨论”本身也很重要。

欢迎讨论玩法、难度、提示设计、实现方式与关卡体验，但请尽量注意公开信息的边界。尤其是在 Issue、Discussion、PR 标题或首页可见区域中，不要直接抛出完整答案、下一关路径或关键隐藏点。若确实需要讨论具体解法，建议明确标注剧透提醒，或者使用更克制的表达方式。

项目鼓励交流，不鼓励破坏他人的首次体验。

## 适用范围

这份行为准则适用于与本项目相关的公开互动场景，包括但不限于：

- GitHub Issues
- GitHub Discussions
- Pull Requests
- Commit 留言与代码审阅交流
- 与项目直接相关的邮件沟通

如果某些互动虽然发生在仓库之外，但明确针对本项目参与者，且已经对项目协作环境造成明显影响，也可能被视为本准则的处理范围。

## 执行方式

本项目由作者个人维护，因此行为准则的解释与执行也由维护者负责。

对于不当行为，维护者可能根据具体情况采取以下措施：

- 提醒或劝阻
- 要求修改评论、标题、描述或其他公开内容
- 隐藏、锁定或删除不合适的讨论内容
- 关闭相关 Issue、Discussion 或 PR
- 拒绝后续互动或贡献
- 在必要时封禁相关参与者

处理时会尽量以“维持讨论环境”为优先，而不是扩大冲突。

如果你在项目相关交流中遇到了明显不适当的行为，可以通过以下方式联系维护者：

- 邮箱：`mojavenight@qq.com`

如果问题涉及敏感内容，建议尽量不要先在公开区域继续升级冲突，可以直接邮件联系。

## 最后

这个项目欢迎不同背景、不同水平、不同表达方式的人参与，但前提是彼此都愿意保持基本尊重。

你不需要对所有设计表示认同，也不需要永远语气温和；但请尽量让你的表达有助于问题被解决，而不是让交流本身变成新的问题。
